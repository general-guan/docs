# TCP

TCP/IP 共分为 5 层

越靠上越接近用户，越靠下越接近设备

|            | 协议                                  |                                                                                                  |
| ---------- | ------------------------------------- | ------------------------------------------------------------------------------------------------ |
| 应用层     | HTTP SMTP POP3 FTP TELNET NNTP RCP... | 让通信符合应用程序各个应用程序有各种各样的协议支持                                               |
| 传输层     | TCP UDP                               | 确保发出的数据能够切实到达收件人的应用程序处                                                     |
| 网络层     | IP                                    | 负责将数据送达收件人的计算机不关心送达的数据是否损坏，或者收件人是否收到                         |
| 数据链路层 | Ethernet FDDI ATM PPP PPPoE...        | 确保能够在直接连接于网络上的设备之间传递。为了完全清除网络层与物理层的不同，准备有各种各样的协议 |
| 物理层     |                                       | 负责将数据转换为信号，将信号转换为数据。转换方法因通信媒体不同而不同，所以没有特定的协议         |

一般情况下，TCP/IP 指的是上述 5 层的整体。但有时也指 TCP 和 IP 这两个单独的协议。为了避免混淆，有时将 5 层协议叫做 TCP/IP 协议族。

| 协议名称 | 服务名称       |
| -------- | -------------- |
| http     | WWW            |
| https    | WWW（SSL/TLS） |
| ftp      | 文件转发       |
| mailto   | 电子邮件       |
| telnet   | 远程登录       |
| file     | 本地文件       |
